前幾天文章談過，Nvidia財報說明會宣布其Vera訂單200億美元之後許多人認為是Arm架構優秀的原因，這項市場看法有對ARM "過譽" 之嫌，因為Nvidia不是用Arm標準核心如Neoverse，而是自研的Olympus，有可能是自研核心的優秀，以及產能供應優勢，今天這篇轉貼文評測了工程版Vera CPU，證明Nvidia Olympus核心的確相當優秀，是評測者評測過最佳的Arm ISA陣營CPU，作者提到一句話 "NVIDIA Vera 的效能遠勝於我們所見過的 Ampere Computing 或 Google Compute Engine 和 Microsoft Azure 等公有雲供應商的客製化 ARM 解決方案"

所以，Vera 200億美元訂單，在產品面，確實不是Arm CPU設計優秀的功勞，而是Vera自研CPU IP的功勞，Arm被市場 "過譽" 了，Vera遠勝其他使用Arm標準CPU核心CPU(包含Nvidia上一代的Grace CPU)

ARM ISA: 作業系統、軟體、生態系，大家一樣
ARM CPU IP: Nvidia自研、其他人大部分是授權ARM標準CPU IP

Vera CPU產品性能上足以和x86 CPU抗衡，也是市面上唯一可以和x86抗衡的Arm CPU，如果以往效能不如Vera的Arm ASIC/ASSP CPU可以靠著省電和生態系能贏得x86的市佔率，那Vera CPU就能贏得更多

----------"NVIDIA Vera CPU 效能測試：自研Olmypus核心展現ARM史上最佳效能
.......
NVIDIA 的 Vera CPU 採用自家設計的 Olympus CPU 核心，展現出與 Intel/AMD x86_64 CPU 競爭的強大實力，這是我從未見過其他 ARM 或非 x86_64 處理器的表現。繼續閱讀這些 Linux 上 NVIDIA Vera CPU 的早期基準測試。
.......
與使用 Arm Neoverse-V2 核心的 NVIDIA Grace 不同，Vera 採用 NVIDIA 自家的「Olympus」核心設計。
......
Vera 配備 88 顆 Olympus 核心，聲稱能提供前代兩倍的性能與領先的能源效率。Olympus 相容於 Armv9.2 指令集架構架構，支援 FP8 精度，透過空間多執行緒共支援 176 執行緒，並搭配 LPDDR5X 記憶體，提供高達 1.2TB/s 的記憶體頻寬。在 Grace 之上，Vera 擁有雙倍的 L2 快取，每核心 2MB，統一 L3 快取容量更大，達 164MB，並支援 PCIe Gen 6 及 CXL 3.1 連線。

Vera CPU 在這次初步測試中測試時，插槽 TDP 峰值為 450 瓦。加上LPDDR5X記憶體，功耗大約是 50 瓦或更少。
......
Linux 7.1+ 已經支援了關鍵驅動程式，Vera 應該能在 ARM64 伺服器上的 Linux 發行版如 Ubuntu、Fedora 等運作。NVIDIA 也會繼續提供基礎作業系統作為他們修改版的 Ubuntu 版本，並已準備好所有 Vera 補丁。NVIDIA Vera 在 Linux 上是依賴 ACPI，無需處理令人沮喪的裝置樹或其他麻煩。
......
我沒能測試在 NVIDIA Vera 或類似平台載入不同 Linux 發行版，但從我在 NVIDIA 的經驗來看，Vera 的上游開源 Linux 支援已經相當不錯。我的測試是在 Ubuntu 24.04 LTS 上進行的，他們的基礎作業系統配置是經過修補的 Linux 6.18 LTS 核心，搭配 GCC 16.1。

搭載 Olympus 核心的 NVIDIA Vera 在 Linux 上運作得相當順暢。有一點需要注意的是，部分電力管理的調整還在上游階段。最近我介紹了 NVIDIA 在 Linux 上開發 ACPI CPPC v4 支援，這似乎是相關的部分之一。由於電源管理調整仍在進行，不幸的是，NVIDIA 要求在這輪初次基準測試期間不要啟動 CPU 功耗監控。同樣地，在這第一輪測試中，CPU 頻率監控也未被允許。此外，NVIDIA Vera 的基準測試是從他們的預生產開放平台系統進行，而這對於 2026 年稍晚實際量產封閉式伺服器機殼設計的功率與頻率評估將更具參考價值。
......
但根據這些數據相較於 EPYC/Xeon，很容易說 Vera 是我測試過最具競爭力的 ARM 伺服器 CPU，無論是裸機還是公有雲。
......
(中間一大堆各種benchmark評測)
......
在這輪 NVIDIA Vera 基準測試中允許的多種工作負載中，Vera 超出了我預期，從未見過 ARM64 處理器能如此強力地與 x86_64 競爭。以地平平均計算，NVIDIA Vera 的效能比 AMD EPYC 9575F 5.0GHz 高頻處理器高出 10%。以Gen-on-Gen而言，Vera的效能幾何平均值是1.63倍。在英特爾目前旗艦 Granite Rapids 處理器中，搭載單一顆 Intel Xeon 6980P 的 NVIDIA Vera 提供了 1.55 倍的效能。

我其實不太清楚 NVIDIA Vera 搭配新 Olympus 核心會有什麼期待。但最後我意識到，這是有史以來對 Intel 和 AMD x86_64 處理器最強大的競爭對手。NVIDIA Vera 的效能遠勝於我們所見過的 Ampere Computing 或 Google Compute Engine 和 Microsoft Azure 等公有雲供應商的客製化 ARM 解決方案。
......
NVIDIA Vera 在多種工作負載上，實際上能與最新的 AMD EPYC 和 Intel Xeon 處理器競爭。當然，NVIDIA 將初期基準測試的範圍限制在目標市場和應用案例，但至少它足夠多元，讓我們能做出一些廣泛的實現，而這些工作負載大多是我在其他基準測試文章中多年來一直在執行的。今天的結果只是我典型偏好的飽和轟炸基準測試方法中的一小部分，考慮到當今多元的工作負載和廣泛的用戶/讀者興趣，但希望隨著 Vera 今年晚些時候的加強，我們能以此作為樂趣。

NVIDIA Vera 在功耗方面應該相當有趣。Vera CPU 的 TDP 為 450 瓦，而 LPDDR5X 記憶體只有約 50 瓦，且記憶體頻寬非常驚人。所以只有 500 瓦，而頂尖的 AMD EPYC Turin 和 Xeon Granite Rapids CPU 本身 TDP 就只有 500 瓦。DDR5 ECC RDIMM 記憶體或搭載 Xeon Granite Rapids 的 MRDIMM 在載入全部 12 個記憶體通道時，功耗可遠超過 50 瓦......關於 Xeon 6 的一些 MRDIMM/DDR5 記憶體功率（及效能）數據，請參閱先前的 Xeon 6 文章。因此，整體「壁掛式電力」數據對 Vera 來說應該是勝利，且能更有效率，但目前沒有數據可分享。正如我們從 Stream 記憶體基準數據中看到的，LPDDR5X 記憶體甚至超越了 Granite Rapids 的 MRDIMM。
......
根據現有的測試，這絕對是我測試過效能最高的ARM Linux伺服器處理器。而Phoronix不到兩週就要滿22歲了。我一直在 ARM Linux 上做效能基準測試，從 Calxeda 伺服器時代開始，還有這些年來其他有趣的改編，比如垃圾桶裡的太陽能驅動 ARM 叢集。在測試的工作負載中，搭載 Olympus 核心的 NVIDIA Vera CPU 證明其競爭力可與現行世代的 AMD EPYC 9005「Turin」處理器媲美。NVIDIA Vera 幾乎全面擊敗頂尖的 Intel Xeon 6980P「Granite Rapids」處理器，即使 Xeon 6980P 搭配 MRDIMM-8800 記憶體。
......
NVIDIA Vera 的上市時機與定價也將扮演重要角色。Vera 今年晚些時候會加強。基於Zen 6的AMD EPYC Venice也預計今年推出，但具體時間與量產量程尚未公開。EPYC Venice 預計將對 EPYC Turin 帶來顯著漲幅，因此一旦加速，他們可能會領先。同時，Intel Xeon Diamond Rapids 預計會在 2027 年某個時候推出。透過 AMD EPYC Venice，它很可能會在 Vera 設計的工作負載之外，搶佔多元工作負載的領先地位，但更重要的是 NVIDIA 能多快地推出 Vera 的後繼者。如果 NVIDIA 持續積極推動硬體發佈週期，並能比 AMD 約兩年一次 EPYC 發佈間隔更快創新，若能在 Olympus 這優良基礎上快速發展，將能帶來顯著且適時的效能領導。

目前價格仍是未知數，NVIDIA Vera 在超大規模業者、AI 公司及其他主導業者之外，近期可用的情況如何。Vera 的提升過程以及未來幾個月伺服器價格都非常值得關注。
