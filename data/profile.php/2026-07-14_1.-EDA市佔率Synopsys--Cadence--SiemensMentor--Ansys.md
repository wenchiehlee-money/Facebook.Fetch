------------"流程始於以 Verilog、SystemVerilog 或 VHDL 撰寫的 RTL。邏輯綜合會利用程序專用的標準單元函式庫，將此 RTL 映射到閘級網表。合成引擎必須在時序、面積、功耗及可測試性限制下優化布林邏輯，同時尊重製程、電壓及溫度角落的函式庫特性。對於高效能設計來說，合成並非單一遍次。工程師會迭代約束、時脈定義、階層邊界、重新定時選項及物理感知設定，以減少臨界路徑延遲，然後再進行實體實作。The flow begins with RTL written in Verilog, SystemVerilog, or VHDL. Logic synthesis maps this RTL into a gate-level netlist using a process-specific standard-cell library. The synthesis engine must optimize Boolean logic under timing, area, power, and testability constraints while respecting library characterization across process, voltage, and temperature corners. For high-performance designs, synthesis is not a single pass. Engineers iterate constraints, clock definitions, hierarchy boundaries, retiming options, and physical-awareness settings to reduce critical-path delay before physical implementation.

實體設計接著將網表轉換為佈局。樓層規劃定義了巨集佈局、電網、時鐘拓撲、電壓島及路由資源。投放引擎能定位數百萬甚至數十億個實例，同時盡量減少線長、擁塞、時序違規及功率密度。時脈樹合成插入緩衝區並平衡時脈域間的偏移。銑削工具會在多層金屬間分配互連，同時遵守間距、寬度、通孔、電遷移、多重圖案及密度規則。在前沿節點，布線受數千項設計規則限制，局部佈局決策可同時影響良率、時序、紅外線損失及訊號完整性。Physical design then converts the netlist into layout. Floorplanning defines macro placement, power grids, clock topology, voltage islands, and routing resources. Placement engines position millions or billions of instances while minimizing wirelength, congestion, timing violations, and power density. Clock-tree synthesis inserts buffers and balances skew across clock domains. Routing tools assign interconnect across many metal layers while obeying spacing, width, via, electromigration, multi-patterning, and density rules. At leading-edge nodes, routing is constrained by thousands of design rules, and local layout decisions can affect yield, timing, IR drop, and signal integrity simultaneously.

簽約分析驗證實體實作在電氣上是穩健的。靜態時序分析檢查多種PVT情境下的建立與保持閉合，提取寄生元件、時脈不確定性及片上變異模型。寄生抽取會從最終布線佈局中計算電阻、電容及耦合效應。電力完整性工具評估動態與靜態紅外線下降、電遷移及局部電流密度。訊號完整性分析會檢查串擾引起的延遲偏移與雜訊。這些分析相互依存：固定時序會加劇擁擠，減少紅外線下降可擴大面積，改變路由則可能改變寄生元件，重新開啟時序違規。Signoff analysis verifies that the physical implementation is electrically robust. Static timing analysis checks setup and hold closure across many PVT scenarios, extracted parasitics, clock uncertainties, and on-chip variation models. Parasitic extraction calculates resistance, capacitance, and coupling effects from the final routed layout. Power integrity tools evaluate dynamic and static IR drop, electromigration, and local current density. Signal-integrity analysis checks crosstalk-induced delay shifts and noise. These analyses are interdependent: fixing timing can worsen congestion, reducing IR drop can increase area, and changing routing can alter parasitics enough to reopen timing violations.

功能驗證通常是工程中最大的工作負載。模擬會針對RTL及閘級模型執行有向與約束隨機測試。形式驗證可證明 RTL 與合成網表的等價性，檢查協定屬性，並驗證不可達狀態的假設。覆蓋工具追蹤已演練狀態、轉換、斷言及功能情境。對於大型 SoC，硬體模擬會將設計映射到專用硬體上，讓韌體、驅動程式、作業系統和系統工作負載能在矽片回歸前運行。這對於 AI 加速器與資料中心處理器至關重要，因為全端驗證需要計算陣列、記憶體控制器、互連結構、PCIe、CXL、HBM、安全引擎及軟體執行層之間的互動。Functional verification is usually the largest engineering workload. Simulation runs directed and constrained-random tests against RTL and gate-level models. Formal verification proves equivalence between RTL and synthesized netlists, checks protocol properties, and validates unreachable-state assumptions. Coverage tools track exercised states, transitions, assertions, and functional scenarios. For large SoCs, hardware emulation maps the design onto specialized hardware so firmware, drivers, operating systems, and system workloads can run before silicon returns. This is essential for AI accelerators and datacenter processors, where full-stack validation requires interactions among compute arrays, memory controllers, interconnect fabrics, PCIe, CXL, HBM, security engines, and software runtime layers.

實體驗證是最終的製造門檻。設計規則檢查用以驗證是否符合鑄造廠幾何限制。佈局與原理圖比較證明遮罩佈局實現了預期電路。天線檢查、密度檢查、光刻感知檢查及可靠性檢查可降低製造及終生故障風險。最終輸出為GDSII（或OASIS）資料庫，送至晶圓廠用於掩膜產生與晶圓製造。Physical verification is the final manufacturing gate. Design rule checking verifies compliance with foundry geometry constraints. Layout-versus-schematic comparison proves that the mask layout implements the intended circuit. Antenna checks, density checks, lithography-aware checks, and reliability checks reduce manufacturing and lifetime-failure risk. The final output is a GDSII or OASIS database delivered to the foundry for mask generation and wafer fabrication.
