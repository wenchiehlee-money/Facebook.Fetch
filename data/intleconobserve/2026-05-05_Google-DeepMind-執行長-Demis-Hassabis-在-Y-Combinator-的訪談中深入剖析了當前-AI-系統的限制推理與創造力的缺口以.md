Google DeepMind 執行長 Demis Hassabis 在 Y Combinator 的訪談中，深入剖析了當前 AI 系統的限制、推理與創造力的缺口，以及他對未來幾年技術走向的判斷。

#AGI還缺少哪些核心能力

Hassabis 認為目前的技術組件，包括大規模預訓練、RLHF、思考鏈推理，幾乎可以確定會是最終 AGI 架構的一部分。「我看不到一個世界，會在幾年後突然發現這些路線是死胡同，那對我來說毫無道理。」

但他也明確指出仍未解決的關鍵問題：「持續學習、長期推理、某些記憶層面，這些都還沒解開。還有如何讓系統在各個面向都更一致。我認為這些對 AGI 都是必要的。」

對於是否還需要全新的突破性想法，他保留可能性：「現有技術可能透過一些創新和漸進式創新就能擴展到那個層次，但也可能還缺一兩個大想法需要被攻克。如果真有，我不認為超過一兩個。我的賭注大概是五五波。」

#記憶與ContextWindow的根本問題

談到目前處理記憶的方式，Hassabis 直言：「我們現在算是用膠帶把所有東西黏在一起，全部塞進 context window，這有點不太令人滿意。」

他解釋為何單純擴大 context window 並非答案：「即使我們處理的是機器而非生物大腦，理論上你可以擁有上百萬甚至上千萬個 token 的 context window 而且完美無誤，但查找和找出真正與當下決策相關內容的成本依然不容忽視。即使能全部儲存，這個成本也是非同小可。」

他用人類認知做對比：「context window 大概等同於工作記憶。人類只有幾位數的工作記憶，平均約七位數，而我們有百萬甚至千萬的 context window。但問題是我們把所有東西都塞進去，包括不重要的、錯誤的東西。這相當粗暴，看起來不對。」

具體的瓶頸在處理連續資料時更明顯：「如果你要處理直播影片，天真地把所有 token 記錄下來，一百萬其實不算多，大概只有 20 分鐘。如果你想理解使用者一兩個月內生活中發生的事情，需要的更多。」

#為什麼AI仍會犯基本推理錯誤

Hassabis 指出思考範式還有大量創新空間：「我們現在做的是相當簡單、相當粗暴的方法。可以想像在監控思考鏈、甚至中途介入思考過程上有很大的發揮空間。」

他常和 Gemini 下西洋棋來測試：「所有領先的基礎模型都不太擅長遊戲，這滿有趣的。觀察思考軌跡很酷，因為這是可以被理解和驗證的，我能很快判斷它是否走偏。」

他描述他觀察到的具體缺陷：「我們看到的是，有時它考慮一步棋，意識到那是失誤，但找不到更好的，於是又回去走那步。在一個精確推理系統裡，根本不該看到這種事。」

這就是所謂的「鋸齒狀智慧」現象：「一方面它能解決 IMO 金牌等級的問題，另一方面如果你用某種方式提問，它仍會犯基本的數學錯誤或基本推理錯誤。對我來說，似乎少了某種對自身思考過程的內省。」

#持續學習：阻礙 agent 發揮全部潛力的關鍵

「目前沒有持續學習，是阻礙 agent 執行完整任務的因素之一。它們對任務的某些面向真的很有用，你可以拼接它們做出很酷的事情，但它們無法很好地適應你所處的情境。」

Hassabis 認為這是讓 agent 真正能自主運作的缺失環節：「它們需要能夠學習你即將投入的特定情境。我們必須攻克這一點才能達到完整的通用智慧。」

對於目前 agent 的成熟度，他給出冷靜的評估：「我們才剛開始。過去幾個月才真的開始找到有價值的應用場景，而技術可能也只是剛達到那種程度，不再是好玩的玩具展示，而是真的為時間和效率增加價值。」

他對某些做法保持懷疑：「我看到很多人放出幾十個 agent 跑 40 小時，但我還沒看到產出能合理化那種程度的投入。我認為我們仍處於實驗階段。我們還沒看到一款由 vibe coding 產出的 AAA 遊戲登上應用商店排行榜。」

#A創造力的天花板

Hassabis 用 AlphaGo 的第 37 手作為起點，說明目前 AI 創造力的限制：「對我來說，那只是有趣、很有用，但它能否發明圍棋本身？」

他描述他想要的系統能力：「給它一個高層級描述，一個你能在 5 分鐘內學會規則、但需要好幾輩子才能精通的遊戲，美學上美麗，但你能在一個下午幾小時內就能玩。也許你能想像那是我會給的高層級描述。然後我希望它回傳的東西是圍棋。今天的系統顯然做不到這點。」

他保留另一種可能性：「也許在座有人能做到，那答案就是什麼都不缺，只是我們使用系統的方式問題。也可能今天的系統就有這能力，只要有夠聰明、有創造力的人來使用，並提供那個推動力，那個專案的靈魂，並能與工具熟稔到幾乎與工具合一。」

#AI能做出真正的科學發現嗎

對於 AI 是否能進行真正的科學推理，Hassabis 保持審慎樂觀：「我認為快了。我們有 co-scientist 系統，還有 AlphaEvolve 等演算法，能稍微超越基礎 Gemini 的能力。所有前沿實驗室都在這方面實驗。但目前我還沒看過任何真正、巨大的發現，這是我個人的看法。我認為它正在到來。」

他認為突破與創造力以及超越已知邊界有關：「在那個層次上顯然不只是模式比對，因為沒有模式可比對，也比外推法更多一點，是某種類比推理。我不認為這些系統具備那個，或至少我們沒以正確的方式使用它們。」

他提出一個更高的標準：「能不能想出一個真正有趣的假設，而不只是解決一個假設？我們現在在說的『只是』是像解決黎曼猜想或某個千禧年大獎難題。我最喜歡的是 P 等於 NP。但比那更難的是，能否提出一組新的千禧年問題，被頂尖數學家視為深刻、有意義、值得花一生去研究和努力解決的問題。」

#推理成本的物理極限

Hassabis 對推理成本歸零的願景持保留態度：「我不確定推理成本會真的變成幾乎免費。傑文斯悖論告訴我們，所有人最後都會用上手邊能拿到的所有運算資源。」

他預測未來運算需求的方向：「你可以想像數百萬個 agent，agent 群體一起工作。或者單一 agent、較小的 agent 群體在多個方向思考然後集成。所有這些都會用光所有可用的推理資源。」

關於能源層面的瓶頸：「也許某天能源成本可以接近零。如果我們解決核融合、超導體、最佳電池等問題，我認為材料科學會幫我們解決，能源成本會基本上是零。但仍會有晶片的物理製造和其他瓶頸，至少在未來幾十年內如此。所以推理側仍會有配給，仍必須有效率地使用。」

#小模型的潛力上限

對於小模型能變多聰明，Hassabis 認為還有很大空間：「我不認為我們已經到達任何資訊密度的極限。也許某天會碰到。但目前我們的假設是，旗艦版 Pro 或前沿模型推出後半年到一年，你就能在非常小的、幾乎邊緣端的模型上看到這些能力。」

他描繪未來邊緣運算的樣貌：「在邊緣端運行這些模型，除了效率，也為了隱私和安全。如果你想到不同裝置會處理非常個人的資訊，包括家用機器人，你會想要非常有效率、非常強大的本地模型，搭配雲端的更大模型來協調，只在特定情境下委派給雲端。例如所有音訊和視覺串流可能都在本地處理，並且只留在本地。我可以想像那會是非常好的最終狀態。」

#虛擬細胞：未來 10 年的科學突破

Hassabis 描繪了生物學領域的長期願景：「最終你想要一個完整的虛擬細胞，一個運作中的細胞完整模擬，你可以擾動它，輸出結果與實驗結果夠接近以致有用。你能省略很多搜尋步驟，產生大量合成資料來訓練其他能預測真實細胞行為的模型。」

時間表上他給出明確估計：「我認為距離某種完整虛擬細胞大概 10 年。我們在 DeepMind 的科學側從虛擬細胞核開始，因為它相對自包含。」

他點出當前最大的瓶頸是資料：「目前資料還不夠。如果我們能對活細胞成像而不殺死細胞，會是顛覆性的突破，你就能把它轉換成視覺問題，那是我們知道怎麼解的。但目前我不知道有任何技術能在不破壞活的、動態的細胞下，提供奈米級解析度，看到所有交互作用。」

#未來幾年的科學突破預期

Hassabis 認為科學各領域正處於關鍵時刻：「我看不到任何科學或工程領域，AI 會無法提供幫助。材料科學、藥物發現、氣候建模、數學，我們現在幾乎處於一個 AlphaFold 1 的時刻。結果非常有希望，但還沒完全解決那個領域的大挑戰。未來幾年我們會在所有這些領域有很多可談的。」

關於 AlphaFold 帶來的影響：「全球超過 300 萬研究人員，幾乎每位生物研究人員都在使用 AlphaFold。我的藥廠高層朋友告訴我，從現在起幾乎每一個被發現的藥物，在藥物發現過程中某個階段都會用過 AlphaFold。」

#AGI時間表與未來架構

Hassabis 的 AGI 時間表落在 2030 年左右。他預測未來的系統架構不會是單一的巨大模型：「我能想到 Gemini、Claude 或這些通用系統會把 AlphaFold 這樣的專門系統當成工具使用。

我不認為一切都會放在一個巨大的大腦裡，因為會有太多的回歸，如果我把所有蛋白質塞進 Gemini，那毫無道理。我們不需要 Gemini 做蛋白質摺疊。」

他點出資訊密度的取捨：「回到資訊效率的問題，這肯定會以不好的方式影響它的語言能力。更好的方式是擁有真正優秀的通用工具使用模型，它們甚至可能訓練那些特定工具，但工具本身會在獨立系統中。」

#接下來6到12個月的關鍵觀察點

對於 vibe coding 和 agent 應用，Hassabis 給出明確的時間預期：「為什麼還沒有小孩用這些工具做出賣 1000 萬份的熱門遊戲？這應該是可能的，但還是缺了什麼。也許和流程或工具有關。我預期這在未來 6 到 12 個月內會出現。」

他描繪這個過渡期會看到的景象：「我們不會先看到自主運作。我們會看到這個房間裡的人以 1000 倍的效率運作，那才是你會先看到的。然後其中很多人會建立遊戲公司或其他類型的公司，用這些工具做出某種暢銷 app 或暢銷遊戲。那是你會先看到的，然後更多會被自動化。」
