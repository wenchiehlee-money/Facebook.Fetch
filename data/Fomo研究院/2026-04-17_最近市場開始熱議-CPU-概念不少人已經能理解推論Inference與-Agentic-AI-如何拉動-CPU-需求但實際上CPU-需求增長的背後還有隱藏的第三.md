最近市場開始熱議 CPU 概念，不少人已經能理解推論（Inference）與 Agentic AI 如何拉動 CPU 需求。但實際上，CPU 需求增長的背後還有隱藏的「第三股力量」。
​
這股力量所產生的需求，與前兩者有本質上的不同。
​
前兩者的需求發生在「GPU 機架之內」，為了配合 GPU 運作，CPU 的配比從過去的 1:8 一路飆升到現在的 1:2 甚至 1:1。
​
然而，這第三股力量卻是發生在「GPU 機架之外」，它正迫使科技巨頭們大規模部署完全不帶 GPU 的「純 CPU 機架」。
​
要理解 2026 年的算力佈局，我們必須看清這三股推動力如何從量變走向質變：
​
▋第一股力量：推論時代（Inference Scaling）的全面降臨
​
2026 年 3 月的 GTC 大會上，黃仁勳宣告了「推論時代」的到來。這意味著 AI 算力的消耗重心，已正式從實驗室裡的「模型訓練」轉向現實世界的「用戶服務」。
​
訓練模型是「蓋工廠」，蓋一次就好；但推論是「工廠 24 小時接訂單」。當全球 AI 推論量從每天幾百萬次暴增到數十億次時，CPU 負責的「打雜工作」：接收請求、任務排隊、Tokenization 以及結果格式化，其總量也隨之呈現指數級爆炸。
​
即使單次請求的負擔不重，但當「訂單量」大到一定程度，負責調度的 CPU 就成了嚴重的瓶頸。這也是為何 Lisa Su 在年初強調，推論需求的增長直接帶動了 CPU 需求的顯著上升。
​
▋第二股力量：Agentic AI，從「直線」到「循環」的質變
​
如果說推論時代是「量」的爆炸，Agentic AI（代理型 AI）則是「質」的翻轉。
​
以前的 AI 是單向的：用戶提問，GPU 計算，給出答案。但現在的 Agentic AI 在執行任務（如規劃行程、訂票）時，會進入一個多步驟的「循環」（Loop）。
​
在這個循環中，GPU 只負責短暫的思考，而其餘 50% 到 90% 的工作量，包括呼叫工具、邏輯判斷、處理報錯、管理狀態，全壓在 CPU 身上。
​
這導致了第一種 CPU 需求：GPU 機架內部的「領班」。
​
為了不讓昂貴的 GPU 在等待指令時閒置發呆，資料中心必須大幅提升 CPU 與 GPU 的配比。從過去的 1:8，到現在如 Vera Rubin NVL72 架構中看到的 1:2 甚至 1:1。
​
▋第三股力量：RL 訓練與合成資料
​
這是一個很少人提到的變數，但卻十分重要。
​
這股力量發生在 GPU 機架之外，催生了第二種 CPU 需求：純 CPU 組成的伺服器。
​
到了 2025 年底，人類高品質數據已被 AI 讀光，AI 要進化不能再只靠「背書」，必須靠「自己跟自己練習」。這就是強化學習（RL）與合成資料的崛起。
以前的預訓練像是讓 AI 坐在那裡「讀完一整座圖書館」。這種純數學的矩陣運算，完全是 GPU 的天下。但現在的強化學習，更像是教一個學徒修車或訂機票，AI 必須親自「動手」：嘗試點擊按鈕 → 觀察環境變化 → 修正錯誤再試一次。
​
為了讓 AI 練習，我們必須為它蓋出數萬個虛擬環境（環境模擬器）。這些模擬器充滿了「If/Else」的邏輯分支與隨機報錯，是標準的「一般軟體」。面對這種錯綜複雜的邏輯任務，擅長算數學的 GPU 跑起來極慢，而身為「邏輯處理大師」的 CPU 卻如魚得水。
​
為了加速進化，前沿實驗室會同時開啟 10,000 到 100,000 個平行的虛擬環境。這 10 萬個虛擬世界需要龐大的 CPU 核心擔任「考場管理員」與「裁判」，盯著 AI 的動作並給予回饋，再將結果丟回給 GPU 更新模型。
​
在 2026 年的新分工裡，GPU 是運動員的「肌肉」，負責思考與發力；而 CPU 則是「練習場與教練」，負責打造環境。 以前運動員只需看錄影帶學習，CPU 需求極低；現在要下場跑幾百萬次，沒有足夠的 CPU 蓋出練習場，AI 的能力就會停滯。
​
這解釋了為什麼 OpenAI、xAI 與 Meta 等實驗室正瘋狂搶購純 CPU 伺服器。SemiAnalysis 在 2026 年 2 月指出：「CPU 缺口已成為制約 RL 訓練進度的最大瓶頸。」而 NVIDIA 推出單機架塞滿 256 顆 CPU 的 Vera 架構，正是為了同時運行數萬個並行 RL 環境而誕生的。
​
▋雙重 CPU 需求的時代
​
現在的CPU算力佈局已經非常明確：
​
1. GPU 機架內： 需要強大的 CPU 處理 Agentic AI 的邏輯調度。
​
2. 純 CPU 機架： 需要海量的 CPU 核心作為 RL 訓練的基礎設施。
​
推論時代讓請求變多，Agentic AI 讓單次請求的 CPU 工作量變大，而 RL 訓練則開闢了純 CPU 運算的新戰場。這三者相乘，解釋了為什麼 CPU 需求的爆炸性成長，遠超出了市場早先的預期。
​
