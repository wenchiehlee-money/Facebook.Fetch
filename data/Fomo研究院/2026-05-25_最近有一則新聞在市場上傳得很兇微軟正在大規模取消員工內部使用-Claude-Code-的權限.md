# 最近有一則新聞在市場上傳得很兇：微軟正在大規模取消員工內部使用 Claude Code 的權限。

最近有一則新聞在市場上傳得很兇：微軟正在大規模取消員工內部使用 Claude Code 的權限。
​
「看吧！連微軟這種萬億級巨頭都嫌貴、用不起了，AI 泡沫果然要破了！」，你可能會立刻得出這個結論。
​
不過，這則新聞背後，藏著三層完全不同的故事，我來為大家解讀一下。
​
▋第一層：一場「功高震主」的內部實驗
​
故事要從 2025 年底說起。當時微軟抱著「 benchmarking（基準測試）」的心態，在內部開放了數千個 Claude Code 的授權，讓自家工程師拿它來跟自家的 GitHub Copilot 進行實測對比。
​
結果尷尬了。
​
Claude Code 憑藉強大的 Agentic（代理化）自動化編碼能力迅速爆紅，工程師們用得欲罷不能，甚至直接把微軟親生的 Copilot 晾在一邊。
​
微軟在內部備忘錄中坦言：
​
「我們同時提供 Copilot CLI 和 Claude Code 是為了做基準測試……但 Copilot CLI 給了我們一個更重要的東西：一個我們可以和 GitHub 一起直接形塑的產品。」
​
翻成白話：「Claude Code 很好用，但我們不能把自家幾萬個工程師的生產力，長期綁在競爭對手的工具上。」
​
這邏輯非常微軟。微軟一邊在外面賣 Copilot，一邊讓自家工程師用對手的工具，在戰略上是站不住腳的。這一步遲早要走，它無關泡沫，只關乎微軟要保護自己的 Copilot 生態系。
​
▋ 第二層：「成本 + 財年結尾」
​
但 The Verge 報導的幾個細節，讓事情變得更有意思：
​
- 6 月 30 日這個截止日，剛好是微軟財年的最後一天。
​
- 取消授權，被內部形容為「砍掉一些營運費用的簡單方法」。
​
- 微軟一度成為 Anthropic 內部使用量最大的客戶之一。
​
官方說的是「戰略」，但時間點透露的是「錢」。
​
而真正讓「Token 太貴」這個說法具有殺傷力的，不是微軟，是 Uber。
​
媒體披露，Uber 的 CTO Praveen Neppalli Naga 在接受採訪時承認，因為在內部推廣了 Claude Code，他們竟然在 2026 年 4 月，就直接把整整一年的 AI 預算給燒光了。
​
在 Uber 內部，高達 84% 的工程師成為這類 Agentic 工具的重度用戶，每人每月消耗的 Token 成本高達 500 到 2,000 美元。
​
當 AI 從「一問一答」的對話框，演變成「自動幫你重構整個專案、在背景瘋狂跑迴圈」的 Agent 時，它消耗 Token 的速度，就像是一台沒有關緊的印鈔機，在瘋狂往外撒錢。
​
這也解釋了為什麼 GitHub 宣布從 2026 年 6 月 1 日起，將 Copilot 改為「按使用量付費（Token-based AI Credits）」，徹底宣告了「無限暢飲」時代的終結。
​
▋ 第三層： AI 進入「成本管理時代」
​
看到這裡，你可能會覺得：既然大家都燒不起，那 AI 是不是要走入死胡同了？
​
事實恰恰相反。
​
Uber 雖然大喊「預算模型得重做（Back to the drawing board）」，但他們並沒有取消 Claude Code 的授權，也沒有限制員工使用。根據外媒報導，儘管預算超支，Uber 依然沒有放慢 AI 推進的步伐，甚至計劃測試 OpenAI 的工具。
​
為什麼？因為生產力的提升是實打實的（甚至有 70% 的提交代碼都有 AI 的參與）。
​
雖然外界分析指出，Uber 的超支是因為初期缺乏像雲端基礎設施那樣的精細化成本控制（如每人限額、即時監控），但 Uber 官方並未宣布任何「限制使用」或「加強治理」的緊箍咒。
​
相反地，CTO 描繪了一個更激進的長期願景：未來將走向「AI Agent 負責編碼、測試與部署，而人類工程師則轉型為協調者（Orchestrators）」的開發模式。
​
這意味著，企業面臨的不是「AI 沒用要放棄」，而是「AI 太有用，以至於舊有的財務預測模型完全跟不上使用速度」。
​
因此，行業正在經歷的是「從 1 到 100」的陣痛：在生產力不倒退的前提下，企業如何重新適應這套全新的計費邏輯？
​
這條路將由三個轉變構成：
​
1. 「AI 財務治理（FinOps for AI）」的興起：
​
過去企業習慣了 SaaS 的「按人頭訂閱」，但 Agent 時代是「按 Token 消耗計費」。未來企業必須學會像管理 AWS 帳單一樣，為不同團隊設置 Token 預算、設定使用上限，並進行嚴格的費用歸因分析。
​
2. 「模型的分層使用」：
​
過去那種無論大事小事，一律丟給最貴、最強的 Frontier Model 的粗放式用法將會消失。未來，簡單的日常任務將由反應快、成本低的小模型（SLM）在背景處理，只有真正遇到架構重構或複雜 Bug 時，才會調用頂級模型。
​
3. 「混合工具鏈」與「自主可控」的護城河：
​
這確立了大型科技公司（Hyperscalers）的新一代競爭優勢。微軟有 Copilot、Google 有 Gemini、Amazon 有 Bedrock，他們擁有自己的模型與平台，能夠在內部消化或轉嫁 Token 成本。而完全依賴第三方 API 的企業，在成本控制上將面臨巨大劣勢。
​
▋ 結論
​
當一個新聞同時可以被解讀成「故事的結束」或「故事的下一章」時，大眾媒體通常會先選擇前者，因為「泡沫破裂」的標題更具煽動性。但理性的決策者，看見的往往是後者。
​
微軟停用 Claude Code，被渲染成 AI 的滑鐵盧；但它真正反映的，是 AI 產業已經大到連微軟、Uber 這樣的巨頭，都必須開始認真算帳了。這是一個產業褪去浮躁、開始走向成熟的里程碑。
​
畢竟，一個學會算帳、懂得追求投資報酬率（ROI）的產業，才是真正能夠落地生根、改變世界的產業。
