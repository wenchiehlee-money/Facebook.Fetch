「在之前，微軟GitHub CoPilot的使用量遙遙領先，但現在其他競爭對手卻成功追上，例如Claude Code，Cursor等等，微軟有甚麼策略嗎？」
​
當Nadella被問到這個問題，看到競爭對手崛起的圖表時，他的回答卻挺有趣：
​
「我很喜歡這張圖...這對我來說是最好的跡象。」
​
競爭對手崛起，但Nadella卻覺得是好事，這好像有點反邏輯？我來為大家拆解。
​
▋GitHub在微軟帝國的定位
​
很多人以為GitHub只是個「放程式碼的地方」，但實際上，它在微軟的AI棋局中扮演著「心臟」和「戰場」的雙重角色。
​
GitHub最大的資產，是開發者們儲存在 GitHub 上的海量程式碼倉庫 (Repositories)，而全世界的軟體開發者基本上都在用 GitHub。
​
在AI時代後，GitHub上出現了各種 AI 編程工具，包括微軟自家的 GitHub Copilot，以及潛在的競爭對手或合作夥伴，如 Anthropic (Claude), Cursor等等。
​
這令GitHub的定位，從過去的存放程式碼，變為 AI 時代軟體開發的中央作業系統。
​
▋佔據開發流程的「心臟地帶」，讓 AI 無所不在
​
開發者的一天通常是這樣：
​
打開 VS Code（微軟的程式碼編輯器）開始寫程式。
寫的過程中，AI 助手 Copilot（微軟的）不斷提供程式碼建議和自動補全。
寫完後，將程式碼存放到 GitHub（微軟的）上。
在 GitHub 上和同事協作、審查程式碼，整個流程由 GitHub Actions（微軟的）自動化。
​
從本地開發到雲端協作，開發者完全沉浸在微軟打造的工具鏈中。AI (Copilot) 不是一個需要額外打開的網站，而是像語法高亮一樣，無縫地融入在整個工作流程中。
​
這創造了巨大的「黏性」。即使有其他單點功能更強的 AI 工具出現，但因為 Copilot 就在「手邊」，開發者更換的惰性成本非常高。
​
微軟不是在賣一個孤立的 AI 工具，而是在升級整個軟體開發的作業系統。通過掌控開發者每天必經的每一個環節，微軟確保了其 AI 服務擁有最優先、最無縫的觸及用戶的權利。任何競爭對手都只是這個作業系統上的一個「應用程式」，而 Copilot 則是「內建核心服務」。
​
▋成為 AI 智能體的「超級聚合者」，化敵為友
​
不過微軟深知，未來不會只有一種 AI。於是，它的策略不是消滅所有對手，而是成為所有 AI 的「調度中心」。
​
GitHub在最近推出了一個名為 「Agent HQ」（智能體總部） 的功能。開發者不再需要自己一行行寫程式，而是可以直接在 GitHub 上提出一個複雜的需求，例如：「幫我建立一個包含用戶註冊、登入和商品展示功能的電商網站後端服務」。
​
這個「總部」會做什麼？它會自動將任務分解，然後調度平台上最好的 AI 智能體去分工協作，最後將所有成果整合在一起，交付給開發者。
​
在這個過程中：
​
- 開發者很開心，因為他用一個統一的介面，就能調動所有頂尖 AI 的能力。
​
- 其他 AI 公司也必須將自己的智能體接入 GitHub，因為這裡有最多的開發者用戶。
​
- 微軟最開心，因為無論開發者最終使用了哪個 AI 的能力，它都掌控著最核心的「任務調度權」和「工作流入口」，並從中收取平台服務費。
​
這是一種「平台聚合」策略。微軟巧妙地將自己從一個「AI 模型參賽者」的角色，提升到了「AI 協作平台主辦方」的角色。
​
它將競爭從「哪個單一 AI 模型最好」的零和遊戲，轉變為「哪個平台能最好地管理和調度所有 AI」的平台遊戲。這讓微軟立於不敗之地：競爭對手的成功，反而會增強微軟平台的生態價值。
​
▋掌握最寶貴的 AI 燃料 — 「程式碼數據」
​
AI 模型需要透過學習海量數據才能變得聰明。對於編程 AI 來說，全世界最高品質、最豐富的學習資料就是人類編寫的真實程式碼。
​
GitHub，作為全球最大的程式碼託管平台，擁有數十億行、跨越幾十年的程式碼。這就是一座無價的「數據金礦」。
​
這形成了一個強大的「飛輪效應」（Flywheel Effect）：
​
1. GitHub 上有海量的程式碼 -> 微軟能用全球最好的數據來訓練 Copilot。
​
2. Copilot 因此變得更聰明、更好用 -> 更多開發者被吸引來使用 Copilot 和 GitHub。
​
3. 更多開發者使用 -> 產生了更多、更新、更高品質的程式碼存放在 GitHub 上。
​
4. 回到第一步，這個正向循環讓飛輪越轉越快，優勢越來越大。
​
數據是 AI 時代最核心的護城河。Google 有全球的搜索數據，Meta 有全球的社交數據，而微軟通過 GitHub 掌握了全球的「程式碼數據霸權」。
​
這個優勢是競爭對手極難複製的。即使對手開發出更先進的模型架構，但如果沒有足夠高品質、大規模的程式碼數據來「餵養」它，其能力也將受限。GitHub 確保了微軟在 AI 編程這條賽道上，永遠擁有最充足的彈藥。
​
▋驅動終極業務 — Azure 雲端計算
​
所有在 GitHub 上運行的 AI 活動，背後都需要消耗巨大的計算資源（算力）。這就像開車需要汽油一樣。
​
- 開發者使用 Copilot 時，每一次程式碼生成請求，都需要在雲端伺服器上進行一次模型推理運算。
​
- GitHub Actions 執行自動化測試和部署時，需要啟動虛擬機器來運行任務。
​
- 未來 Agent HQ 調度大量 AI 協同工作時，更是會產生海量的計算需求。
​
- 訓練下一代 Copilot 模型本身，就需要動用數千張 GPU 進行數週的運算。
​
這些龐大的計算需求，最終都會轉化為對微軟 Azure 雲端服務的訂單。
​
GitHub 不僅是一個存儲代碼的平台，它更是通往 Azure 的巨大流量入口和需求產生器。
​
微軟的戰略是，利用 GitHub 這個前端平台鎖定開發者和 AI 工作負載，然後將這些高價值的計算需求引導到自己的後端基礎設施（Azure）上來變現。這實現了從應用層（GitHub）到基礎設施層（Azure）的完美商業閉環。
​
▋「AI經紀人」與「有線電視套餐」
​
回到Nadella的訪談，我們一起來看看他具體說了甚麼：
​
「有時候我會把它形容成所有這些AI智能體的『有線電視』，因為我基本上會將它們打包到一個訂閱服務中，Codex、Claude、Cognition的東西、任何人的智能體、Grok，它們全都會在那裡。」
​
「猜猜那些生成大量代碼的傢伙們，他們所有的代碼倉庫都放在哪裡？它們都放在GitHub。」
​
他所說的「有線電視」，其實就像Netflix一樣，他們有自家電影，但也會讓其他廠商的電影在平台上播放。你不會在乎電影是誰拍的，你只在乎「Netflix上有沒有好內容看」。
​
就像Netflix需要電影公司的內容一樣，微軟的平台也需要這些AI公司的模型來豐富自己的「頻道列表」。競爭對手的成功，反而增強了微軟平台的吸引力。
