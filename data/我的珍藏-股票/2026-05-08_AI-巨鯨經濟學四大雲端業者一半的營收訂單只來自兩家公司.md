# 【AI 巨鯨經濟學:四大雲端業者一半的營收訂單,只來自兩家公司】🐋

原文連結: https://www.facebook.com/learninvest2020/posts/pfbid09Mg5c7Puee2LgryKN45GT2J9Si5MuSYCNow5DYNfAMp8hraGs44wG2wBhtxh5CK3l
【AI 巨鯨經濟學:四大雲端業者一半的營收訂單,只來自兩家公司】🐋

The Information 攤開一張可能是當前 AI 泡沫辯論最關鍵的圖:美國四大雲端業者的「未實現營收訂單」(Revenue Backlog),有 43-54% 來自 OpenAI 和 Anthropic 兩家公司。整個雲端產業的成長故事,正建立在兩個還沒上市、還在燒錢的客戶身上。

■ 各家集中度(訂單來自 OpenAI + Anthropic 的比例):

・Oracle:$300B / $553B = 54%(全來自 OpenAI,集中度最高)

・Amazon:$238B / $464B = 51%($138B OpenAI + $100B Anthropic)

・Microsoft:$310B / $627B = 49%($280B OpenAI + $30B Anthropic)

・Google:$200B / $467.6B = 43%(全來自 Anthropic)

■ 為什麼「Revenue Backlog」這個指標重要:

・這是雲端業者帳上「已簽約但還沒認列為營收」的訂單

・分析師估值靠這個數字推算未來 5-10 年的成長軌跡

・現在有近一半的數字,壓在兩家私人公司的支付能力上

■ 集中度風險的真實意義:

・OpenAI 與 Anthropic 加起來 $1 兆以上的承諾支出

・但兩家目前都還在燒錢,主要靠 Microsoft、Google、Amazon 的反向投資循環支撐

・形成一個閉環:雲端業者投資 AI 公司 → AI 公司用這筆錢回頭買雲端服務 → 雲端業者把這筆錢認列為「Revenue Backlog」推升估值

這張圖把 AI capex 故事最深層的問題暴露出來:它本質上是一個循環交易結構。

Microsoft 投資 OpenAI $130B,然後 OpenAI 承諾未來付給 Microsoft $280B 的雲端支出。Amazon 投資 Anthropic $40B,Anthropic 承諾付給 Amazon $100B。雲端業者的財報好看,部分是因為他們先把錢借給客戶、再讓客戶用這筆錢回來買服務。

這個結構在多頭情境下沒問題:只要 OpenAI 和 Anthropic 持續成長、能融到下一輪資金,訂單就會兌現,雲端業者的營收就會實現。但任何一個環節斷裂,連鎖反應會非常劇烈:

第一,如果 OpenAI 估值停止上升、新一輪融資受阻,他們無法繼續支付雲端帳單,Oracle 一夜之間失去 54% 的成長故事。第二,如果 Anthropic 成長放緩,Google 失去 43%、Amazon 失去近一半的訂單支撐。第三,這兩家本身的訓練成本還在指數級上升(前幾天 Brandon Carl 那張圖),代表他們需要的雲端支出可能比承諾還多,但收入端的兌現速度沒跟上。

對市場最諷刺的地方在於:Mag 7 的高估值,有相當比例靠這兩家私人公司撐著。當投資人在討論「Mag 7 的 EPS 是不是真的能持續」時,真正該問的是「OpenAI 和 Anthropic 接下來的融資、商業模式、現金流能不能撐住這 $1 兆訂單」。

下一個觀察點是 OpenAI 的下一輪估值與 Anthropic 的 ARR 公布。如果這兩個數字繼續往上走,雲端業者的 backlog 故事就成立;如果其中一家出現任何融資困難或客戶流失訊號,整條 AI capex 鏈會立刻被市場重新定價。
