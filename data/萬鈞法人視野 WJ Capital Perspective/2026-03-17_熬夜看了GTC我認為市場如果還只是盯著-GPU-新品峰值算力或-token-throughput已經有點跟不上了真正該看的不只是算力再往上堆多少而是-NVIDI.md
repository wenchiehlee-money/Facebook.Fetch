熬夜看了GTC，我認為市場如果還只是盯著 GPU 新品、峰值算力或 token throughput，已經有點跟不上了。真正該看的，不只是算力再往上堆多少，而是 NVIDIA 正在把 AI 從模型競賽，推進到 agent 真正落地的基礎設施階段。

黃仁勳在 GTC 2026 直接把 OpenClaw 放到 NVIDIA 生態系裡（就是那隻龍蝦🦞！），官方同步宣布 NemoClaw，主打用單一指令就能把 Nemotron 模型與 OpenShell runtime 拉起來，讓開發者更容易部署可長時間運作、具備安全與隱私控制的 autonomous agents。NVIDIA 官方也明講，現在要做的不只是訓練模型，而是讓 AI agent 能夠被安全地部署、擴展、持續運作。這個方向，不是概念，而是今年 GTC 很清楚的主軸。 ￼

當 AI agent 開始不是做一次性問答，而是持續在線、持續調工具、持續連資料、持續執行任務，背後消耗的就不只是 GPU，而是從先進製程、先進封裝、測試、散熱、電源、互連、光通訊，一路延伸到記憶體與存儲。這不是單點商機，是整串基礎設施重估值。

算力往上堆，本質就是更多電晶體、更高密度、更複雜堆疊。於是先進製程、2nm、1xnm 相關設備與材料、CoWoS、SoIC、CoPoS、SLT、先進封裝耗材、CCL、PCB、建廠，全都不是陪襯，而是核心受惠。因為 AI 伺服器不是只多一顆晶片，而是整個系統級複雜度都在上升。電晶體暴增之後，封裝、傳輸、供電、散熱、測試，每一關都會變成瓶頸，誰卡住，誰就有議價權。

HVDC、大電流、大電壓也是同樣邏輯。電晶體越多，功耗密度越高，資料中心最後一定走向更高效率的供電架構。這不是選配，是大型 AI cluster 成本結構裡愈來愈重要的一段。加上 CPO、矽光子、光電轉換與檢測需求往上，未來真正有技術門檻的，不只是做出產品，而是能不能在高電流、高電壓、高速光傳輸的條件下，把良率、可靠度與檢測一起做到位。這種領域最後通常就是自然寡占。

但這次 GTC 我認為更值得市場重估的，是 AI agent 對記憶體與存儲的拉動。

OpenClaw 這波在全球尤其是中國爆紅，連「養龍蝦」都變成流行語，不是在養真的龍蝦，而是在講部署會自己調工具、跑流程、完成任務的 AI agent。OpenClaw 的 GitHub 熱度非常高，NVIDIA 也公開表示要支援 OpenClaw 生態。這件事的重要性在於，AI 正從「回答問題」走向「持續工作」。 ￼

一旦 agent 變成數位員工，記憶體需求就不能再只看 HBM 或 DRAM。因為 agent 要有短期工作記憶，要接知識庫，要留任務狀態，要保留 context、workflow、cache、snapshot、審計紀錄，還要反覆讀寫向量資料庫。GPU 解決的是算力，HBM 解決的是高頻寬，DRAM 解決的是即時工作集，但大量歷史資料、任務痕跡、長上下文與企業知識，最後不可能都放在最昂貴的那層，勢必要往 enterprise SSD 與 NAND 下沉。

所以我一直講，AI 不是只有 GPU story。agent 真正開始大量進企業之後，最先被重新定價的，不只會是 GPU 供應鏈，也會是整個 memory hierarchy。因為資料中心一定走向分層存儲，高價值、高頻資料留在上層，大容量、低成本、可承接海量 context 與歷史紀錄的，會往高容量 SSD 走。

更關鍵的是，需求在往上走，供給卻沒有同步失控擴張。前幾年 NAND 廠大虧損後，資本支出一直偏保守，這代表如果 AI agent 真的從 demo 變成企業常態工作負載，enterprise SSD 和 NAND 的有效產能會比市場現在想得更容易吃緊。這也是為什麼我現在看 NAND，已經不會只用過去那種庫存循環去看，而是開始用 AI 工作負載結構改變去看。

另外一個容易被低估的是測試。算力越大、封裝越複雜、晶片價值越高，測試時間只會拉長，不會縮短。AI 晶片、先進封裝、探針卡、探針、SLT、系統級驗證，後面都還有很長的需求尾巴。再加上高熱密度下的解熱需求，散熱也不只是搭配題，而是必要條件。沒有散熱，算力根本兌現不了。

所以如果用一句話總結這次 GTC，我的看法很簡單：agent 元年，是整個 AI 基礎設施鏈條再擴張的一年。從先進製程、先進封裝、供電、光通、測試、散熱，到 DRAM、NAND、enterprise SSD，整條線都會被重新拉動。

龍蝦越多，資料越多；資料越多，記憶體和存儲越重要。AI agent 如果真的開始變成企業裡的數位員工，那市場接下來要重估的，就不是單一 GPU，而是整個算力、電晶體、記憶體、存儲與系統基礎設施的總需求。這才是這次 GTC 真正的延伸商機。
